Андрей собрал таблицу с табло аэропорта Б.Савино за период 30.04 – 04.05:

Файл: `docs/savino_manifest_2026-04-30_2026-05-04.xlsx`

### Что есть в нашей БД

ADS-B треки — последовательности позиций (icao с lat/lon) на низкой высоте
у ВПП. Pi3 на 58.0036°N, 55.9397°E (≈ 10 км к С от ВПП USPP — 57.9146°N,
56.0214°E).

Определение «трек» в этом анализе:
- непрерывный отрезок одного icao
- разрыв между точками не более 30 минут (иначе считается новым событием
  — взлёт и посадка одного борта в один день дадут два трека)
- не менее 5 точек
- в радиусе 8 км от ВПП
- altitude < 5000 ft

Тип события (взлёт/посадка) определяется по тренду высоты от первой
к последней точке.

### Методология (почему не считаем по callsign)

Прямой матч `manifest.callsign ↔ positions.callsign` оказался ненадёжен:
у части авиакомпаний оперативный позывной в эфире НЕ совпадает с номером
рейса по табло. Примеры:
- S7 Airlines: commercial `S7 1011` ↔ operational `SBI2626`
- ЮТэйр: `ЮТ 484` ↔ `UTA184`

Для AFL / PBD / SDM / STW / NWS / AZV / UZB / NMA / SHS / LMU обычно
совпадает, для остальных — нет. Поэтому считаем не персональный матч,
а **число событий по обеим сторонам** и сравниваем агрегаты.

### Куда уходят пропавшие 88 событий (~52%)

1. **Clipping АЦП на близких бортах / над зенитом** — слышим как mode_s
   без lat/lon (см. `docs/session_2026-04-28.md`: STW133, PBD6512,

2. **Старые борта без ADS-B Out** — Ту-204, Ил-86, часть региональной

3. **Слепые зоны антенны** — отдельные азимуты или углы места.

Нижнюю границу пункта 2 можно прикинуть отдельным анализом
(сколько из 88 пропавших мы вообще слышали как mode_s), но это уже
после AT-10 — там удобнее сравнивать.

### Прошлая попытка с матчингом по callsign (для истории)

До перехода на агрегатный метод считали 1:1 по позывным с маппингом
авиакомпания → ICAO callsign. Получили 96/168 = 57% ADS-B и 75% по
«доверенным» авиакомпаниям. Метрика завышена за счёт того, что для S7,
ЮТэйр, Азимут, ИрАэро и др. callsign в эфире ≠ номер по табло, и эти
рейсы помечались как «не пойманы», хотя могли быть в эфире под другим
позывным.

Корректный метод — агрегат по «треки vs события на табло». Дал 46%.

### План «после AT-10»

1. Снять `stats.json` ДО установки (если ещё не снят) и ПОСЛЕ.
2. Поставить AT-10 inline антенна → RTL-SDR.
3. Подождать 5 суток — собрать сопоставимый период.
4. Запросить таблицу с табло Б.Савино за этот период.
5. Прогнать SQL ниже — получить новый % захвата.
6. Сравнить с 46%:
   - **Рост до 80–95%** → clipping был доминирующим источником потерь.
   - **Остался 50–60%** → большая часть пропавших без ADS-B Out или
     антенна / геометрия. Дальнейшие меры — другие.

### SQL воспроизведения

```sql
-- ADS-B треки у Б.Савино за период YYYY-MM-DD .. YYYY-MM-DD
WITH near AS (
  SELECT icao, ts, altitude
  FROM positions
  WHERE ts >= 'YYYY-MM-DD 00:00:00+05'
    AND ts <  'YYYY-MM-DD 23:59:59+05'
    AND lat IS NOT NULL
    AND altitude IS NOT NULL
    AND altitude < 5000
    AND sqrt(
          power((lat - 57.9146) * 111.0, 2) +
          power((lon - 56.0214) * 111.0 * cos(radians(57.9146)), 2)
        ) < 8
),
gapped AS (
  SELECT *,
    CASE WHEN ts - lag(ts) OVER (PARTITION BY icao ORDER BY ts) > INTERVAL '30 min'
         OR  lag(ts) OVER (PARTITION BY icao ORDER BY ts) IS NULL
         THEN 1 ELSE 0 END AS new_event
  FROM near
),
numbered AS (
  SELECT *, sum(new_event) OVER (PARTITION BY icao ORDER BY ts) AS event_id
  FROM gapped
),
events AS (
  SELECT icao, event_id,
    CASE
      WHEN (array_agg(altitude ORDER BY ts))[1]
         < (array_agg(altitude ORDER BY ts DESC))[1] THEN 'takeoff'
      ELSE 'landing'
    END AS phase
  FROM numbered
  GROUP BY icao, event_id
  HAVING count(*) >= 5
)
SELECT
  count(*) FILTER (WHERE phase = 'takeoff') AS takeoffs,
  count(*) FILTER (WHERE phase = 'landing') AS landings,
  count(*) AS total_tracks
FROM events;
```
