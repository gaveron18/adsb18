# Сессия 2026-04-15

## Фикс: туннель Pi→prod — "remote port forwarding failed for listen port 30093"

### Симптомы
Страница adsb18.ru загружается, но периодически самолёты не двигаются.
На Pi: `adsb-tunnel-prod.service` — 18 перезапусков за 2 часа после старта.
Ошибка: `Error: remote port forwarding failed for listen port 30093`.

### Причина
`ClientAliveInterval 0` (default) в sshd на prod — мёртвые TCP соединения
не очищались, порт 30093 оставался занят после обрыва туннеля.
autossh не мог переподключиться и ждал 3-5 минут пока TCP сам не освободит порт.

### Что сделано
`/etc/ssh/sshd_config` на 185.221.160.175:
- `ClientAliveInterval 15` (было `#ClientAliveInterval 0`)
- `ClientAliveCountMax 3` (было `#ClientAliveCountMax 3`)
- `systemctl reload ssh` — без обрыва текущих соединений

Теперь мёртвые SSH соединения обнаруживаются за 45 сек → порт освобождается быстро → autossh переподключается с первой попытки.
