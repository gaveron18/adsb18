# Архитектура проекта adsb18

## Обзор

**adsb18** — система сбора, хранения и визуализации данных ADS-B (Automatic Dependent Surveillance–Broadcast). Самолёты транслируют своё местоположение по радио каждую секунду. Приёмник на Raspberry Pi принимает эти сигналы и отправляет на сервер, который хранит историю и показывает воздушную обстановку на карте.

## Компоненты

### 1. Приёмник (Raspberry Pi)

| Элемент | Описание |
|---------|----------|
| Антенна | 1090 МГц (диполь или коллинеарная) |
| RTL-SDR | USB-донгл, принимает радиосигнал |
| dump1090 / readsb | Декодирует ADS-B в SBS-формат, порт 30003 |
| **feeder.py** | Читает SBS с dump1090, пересылает на сервер |

Поддерживается **несколько приёмников** одновременно — каждый подключается независимо и идентифицируется по имени (`AUTH perm-pi1`).

---

### 2. Ingest сервер (`server/ingest/`)

Принимает TCP-соединения от фидеров, обрабатывает поток SBS-сообщений.

**Поток обработки:**
```
SBS строка → sbs_parser.py → SBSMessage
                                  ↓
                          _state[icao] — объединение полей
                          (MSG,1 даёт callsign, MSG,3 — позицию,
                           MSG,4 — скорость, MSG,6 — squawk)
                                  ↓
                          _batch[] — буфер 200 строк или 2 секунды
                                  ↓
                          PostgreSQL (INSERT positions + UPSERT aircraft)
```

**Зачем объединять поля?**
dump1090 шлёт разные типы сообщений отдельно. Один MSG,3 содержит позицию, другой MSG,4 — скорость. Ingest объединяет их в одну запись по ICAO.

---

### 3. База данных (PostgreSQL 16)

#### Таблица `positions` — партиционирована по месяцам

```
positions_2026_03   ← текущий месяц
positions_2026_04   ← следующий (создаётся заранее)
...
```

Хранит **каждое обновление позиции** от всех бортов. Партиции старше 6 месяцев удаляются автоматически.

| Поле | Тип | Описание |
|------|-----|----------|
| ts | TIMESTAMPTZ | Время сообщения |
| icao | CHAR(6) | Hex-адрес борта (уникальный ID самолёта) |
| feeder_id | INTEGER | От какого приёмника |
| callsign | VARCHAR(9) | Позывной (AFL100) |
| altitude | INTEGER | Высота, футы |
| ground_speed | SMALLINT | Скорость, узлы |
| track | SMALLINT | Курс, градусы |
| lat / lon | REAL | Координаты |
| vertical_rate | SMALLINT | Вертикальная скорость, фут/мин |
| squawk | VARCHAR(4) | Код транспондера |
| is_on_ground | BOOLEAN | На земле |

#### Таблица `aircraft` — живое состояние

Содержит последнее известное состояние каждого борта. Обновляется при каждом батче. Используется для быстрой генерации `aircraft.json` без сканирования `positions`.

#### Таблица `feeders` — приёмники

Регистрирует все подключённые Pi: имя, координаты, время последнего подключения, счётчик сообщений.

---

### 4. API (`server/api/`)

FastAPI приложение. Три роли:

**а) Совместимость с tar1090**
- `GET /data/receiver.json` — конфигурация приёмника (нужна при старте фронтенда)
- `GET /data/aircraft.json` — текущие борты в формате tar1090

**б) REST API**
- `GET /api/history?icao=&from=&to=` — трек одного борта за период
- `GET /api/aircraft?hours=24` — все борты за N часов
- `GET /api/feeders` — список приёмников

**в) WebSocket**
- `WS /ws` — пушит `aircraft.json` каждую секунду всем подключённым клиентам

---

### 5. Фронтенд (`frontend/`)

Скопирован из проекта [tar1090](https://github.com/wiedehopf/tar1090). Использует:

- **OpenLayers** — интерактивная карта (OpenStreetMap, другие провайдеры)
- **jQuery** — UI и AJAX запросы
- Раз в секунду запрашивает `/data/aircraft.json` и обновляет карту

Возможности карты: треки, таблица бортов, фильтрация, поиск по позывному/ICAO, отображение высоты цветом.

---

### 6. nginx

Единая точка входа на порту **8098**:

| Путь | Куда |
|------|------|
| `/` | Статика frontend/ |
| `/data/` | FastAPI → aircraft.json, receiver.json |
| `/api/` | FastAPI → REST эндпоинты |
| `/ws` | FastAPI → WebSocket (с upgrade) |

## Поток данных (end-to-end)

```
Самолёт передаёт ADS-B (1090 МГц)
    ↓
RTL-SDR принимает сигнал
    ↓
dump1090 декодирует → SBS строка:
  "MSG,3,1,1,424141,1,2026/03/25,10:23:41,,,10000,,,-55.12,37.45,,,,,"
    ↓
feeder.py пересылает по TCP на :30001
    ↓
ingest: парсит → объединяет по ICAO → буфер
    ↓  (каждые 2 сек или 200 сообщений)
PostgreSQL: INSERT positions, UPSERT aircraft
    ↓
FastAPI: SELECT aircraft WHERE last_seen > NOW()-60s
    ↓
aircraft.json: [{"hex":"424141","alt_baro":10000,"lat":-55.12,...}]
    ↓
nginx → браузер → tar1090 рисует самолёт на карте
```

**Задержка** от сигнала до карты: ~2-4 секунды.

## Протокол подключения фидера

```
Pi  ──────────────────────────────────────────────────▶  Сервер :30001
     TCP connect
     "AUTH perm-pi1\n"               ← идентификация
     "MSG,3,1,1,424141,...\n"        ← SBS поток
     "MSG,4,1,1,424141,...\n"
     "MSG,1,1,1,424142,...\n"
     ...                             ← непрерывно
```

При обрыве связи feeder автоматически переподключается.

## Масштабирование хранилища

| Зона покрытия | Сообщений/сек | За 6 месяцев |
|---------------|---------------|--------------|
| Малая (Пермь) | 10–50 | ~80–400 млн строк |
| Средняя (регион) | 50–200 | ~400 млн – 1.6 млрд |
| Крупная (хаб) | 200–1000 | до 8 млрд строк |

Партиционирование по месяцам позволяет **удалять старые данные мгновенно** (DROP PARTITION) без нагрузки на БД.

cp .env.example .env
# задать POSTGRES_PASSWORD в .env
sudo docker-compose2 up -d
